docker

docker存在一定的虚拟化职能。利用docker不仅可以创建容器，也可以向物理机申请相应的资源，方便程序的迁移与扩展。docker提供了标准的应用打包的方式，形成集装箱式的镜像文件。这个镜像文件中包含了应用所需要的的所有资源与外部引用，同时还描述运行应用所需要硬件上的需求，在部署时还可以灵活的调整。运维工程师使用docker可以方便的对应用进行扩展与还原，得到一个与之前相同的运行环境。

1镜像加速

1.1 开通阿里云容器镜像服务。获取容器加速地址。

1.2 修改daemon配置文件来使用加速器

sudo tee /etc/docker/daemon.json <<-'EOF'
{
"registry-mirrors":["url"]
}
EOF
sudo systemctl daemon-reload
sudo systemctl restart docker

2 docker 常用命令

  • docker pull 镜像名<:tags> – 从远程仓库抽取镜像
  • docker images – 查看本地镜像
  • docker run 镜像名<:tags> – 创建容器，启动应用
  • docker ps – 查看正在运行中的镜像
  • docker stop/start/restart 容器id -停止/启动/重启容器
  • docker rm <-f> 容器id – 删除容器
  • docker rmi <-f> 镜像名: – 删除镜像

注：pull不带tags默认下载最新镜像。-f：参数含义为强制删除，可以删除正在运行的容器与存在容器的镜像。

3宿主机与容器间的通信（端口映射）-p

创建容器的时候添加 -p 宿主机端口:容器应用端口 来添加映射。使外部能通过宿主机的端口来跳转到容器内的应用。

4 后台运行容器应用

创建容器时添加-d来使容器后台运行。

5 容器内部结构

以tomcat容器为例：

Tomcat:lastest 容器

一共三层：

  • 最底层的linux系统，提供基础的linux命令。
  • 在上一层是tomcat运行的基本环境jdk。
  • 最上层才是tomcat应用程序。

docker存在一定的虚拟化职能，在容器部署的时候底层都会有一个迷你的linux系统，只能够支撑容器中应用的运行。也正是由于这个迷你的linux系统使容器获取资源，分配资源得以实现。

6 容器内部执行命令

docker exec [-it] 容器id 命令

  • exec 在对应容器中执行命令
  • -it 采用交互方式执行命令
  • eg:docker exec -it xxxxxxxx /bin/bash 开启一个可以后续操作容器的终端
    • exit 退出容器

7 镜像\容器默认存放路径

centos: /var/lib/docker 容器在containers文件夹中，镜像在image中。

8 容器生命周期

docker生命周期

docker run = docker create +docker start =>runing

docker restart = docker stop + docker start

docker kill 与docker stop 的区别：

  • kill会将容器进程kill掉，重新start后，进程id将改变。
  • stop只会将容器进程停止，进入stopped状态，重新start后，启动停止的进程，所以进程id不变。

docker pause 将容器暂停，挂起容器，可以用docker unpause将容器恢复。

9.Dockerfile 镜像描述文件

​ dockerfile 是一个包含用于组合镜像的命令的文本文档.通过读取dockerfile中的指令按步骤自动生成镜像.

cat > Dockerfile << EOF
## 设置基准镜像,jdk8
FROM java:8
## 说明信息.标明作者等信息
MAINTAINER konghenying
## 切换工作路径. 相当于: cd /
WORKDIR /
## 复制同文件夹下的的web/target目录下以SNAPSHOT.jar结尾的jar包复制到容器内的工作路径下,并命名为app.jar
ADD web/target/*SNAPSHOT.jar app.jar
## 在镜像中执行命令,常见的有设置时区,时间校准,增加字体支持等.安装需要下载网络资源时使用 apk add 
RUN echo "Asia/Shanghai" > /etc/timezone
RUN sed -i 's/dl-cdn.alpinelinux.org/mirrors.ustc.edu.cn/g' /etc/apk/repositories
RUN apk add --update ttf-dejavu fontconfig && rm -rf /var/cache/apk/*
RUN apk add --no-cache tini
## 增加环境变量, 格式 env {变量名} {变量值} . 就像在/etc/profile中写入命令来增加环境变量
ENV LANG en_US.UTF-8
ENV LANGUAGE en_US.UTF-8
## 容器启动之后执行的命令.
ENTRYPOINT ["java","-Djava.security.egd=file:/dev/./urandom","-jar", "-Dcom.sun.management.jmxremote=true", "-Dcom.sun.management.jmxremote.port=33306", "-Djava.rmi.server.hostname=116.62.214.93", "-Dcom.sun.management.jmxremote.authenticate=false", "-Dcom.sun.management.jmxremote.ssl=false","-agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=18311","/app.jar"]
EOF

## 构建镜像 
docker build -t {镜像名称}:{tag} {镜像生成位置}

镜像分层

​ dockerfile中每执行一句命令都将产生一个临时容器.用以保存当前状态.
