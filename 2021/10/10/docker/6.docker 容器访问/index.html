容器访问

宿主机访问容器

前提

  1. dockerfile中将端口暴露.

    EXPORT 80

  2. 启动时将暴露端口映射.

    -p 80:80, -P

分类

宿主机随机端口映射所有端口

docker run -it -d -P redis

宿主机随机端口映射容器指定端口

docker run -it -d -P 6379 redis

宿主机指定端口映射容器指定端口

docker run -it -d -p 6379:6379 redis

宿主机指定ip随机端口映射容器指定端口

#语法 -p 宿主机ip:宿主机端口:容器端口
docker run -it -d -p 10.10.0.2::6379 redis

宿主机指定ip,端口映射容器指定端口

docker run -it -d -p 10.10.0.2:26379:6379 redis

可以分析到:

  1. 5: veth2e262e0@if4: 在宿主机序号5的网卡记录映射到目标为veth2e262e0的虚拟网卡中的序号4的记录上.
  2. 7: veth47940e8@if6:在宿主机序号7的网卡记录映射到目标为veth47940e8的虚拟网卡中的序号6的记录上.
  3. 6: eth0@if7:在容器32480dd13be4中序号6的网卡记录映射到目标为eth0的网卡中的序号7的记录上.
  4. 4: eth0@if5:在容器c782648e59b4中序号4的网卡记录映射到目标为eth0的网卡中的序号5的记录上.

随机端口映射原理

通过命令查看随机端口配置.

root@matebook:~# sysctl -a|grep port|grep range
net.ipv4.ip_local_port_range = 32768    60999

记录显示,linux系统将随机端口范围指定为32768到60999.

docker创建随机端口时,从起始端口开始找.找到没有被占用的端口并使用.

容器间访问

创建网桥

# 命令格式 docker netwrok create -d [bridge|host|none] [netwrokname]
root@matebook:~# docker network create -d bridge kafkaSingle
542cdc3f8f04573296028ab12c98fe731cd701d9e9194530c5b1a5b2c636eef0

容器使用网桥

方式一: 创建的时候指定网络

通过 --netwrok networkname 来指定使用网桥情况. 

docker run -d --name zookeeper -p 2181:2181 --network  kafkaSingle  zookeeper

方式二: 中途添加网络

docker network connect [OPTIONS] NETWORK CONTAINER

OPTIONS

参数 说明
–alias string 为容器添加网络范围的别名,其他容器可以通过别名来访问当前容器 eg:ping zookeeper
–ip string 指定ip地址
–ip6 string 指定IPv6地址
–link list 添加链接到另一个容器
–link-local-ip strings 添加容器的链接本地地址

例子

1. 将容器zookeeper连接/断开到网络kafkaSingle中

docker netwrok [connect|disconnect] kafkaSingle zookeeper

3. 使用容器名来连接

用户自定义的网卡可以在容器之间提供自动的 DNS 解析，缺省的桥接网络上的容器只能通过 IP 地址互相访问，除非使用 –link 参数。在用户自定义的网卡上，容器直接可以通过名称或者别名相互解析。

在启动kafka时配置zk地址使用容器名称,会自动寻找指定网络下的containers中name为zookeeper或者别名为zookeeper的容器.

docker run -d --name kafkaSingle  -p 9092:9092  -e KAFKA_BROKER_ID=0  -e KAFKA_ZOOKEEPER_CONNECT=zookeeper:2181  -e KAFKA_ADVERTISED_LISTENERS=PLAINTEXT://127.0.0.1:9092  -e KAFKA_LISTENERS=PLAINTEXT://0.0.0.0:9092  --network  kafkaSingle  a692873757c0

4. 使用别名来连接

创建一个redis容器,并将容器添加进kafakSingle网络中,并配置别名为redis01.

docker run -d -P --name r1 redis
docker netwrok connect --alias redis01 kafkaSingle r1

通过docker inspect r1 查看其中的网络信息.

可以发现netwroks中多了一块kafakSingle网络.并且Aliases参数中多了刚配置的别名redis01.
